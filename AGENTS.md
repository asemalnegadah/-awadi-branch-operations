# تعليمات العمل على مستودع نظام العوادي – عدن

## المرجع الأعلى

اقرأ قبل أي تعديل:

1. `docs/MASTER_SYSTEM_SPECIFICATION.md`
2. `docs/IMPLEMENTATION_PLAN.md`
3. جميع القرارات داخل `docs/adr/`

## حدود النطاق

- النظام لفرع عدن فقط.
- يمنع إضافة `branch_id` أو `tenant_id` أو منطق تعدد الفروع.
- يمنع إدخال بيانات عملاء حقيقية أو أسرار أو مفاتيح إلى المستودع.
- لا تُنفذ حركة مالية أو تعديل رصيد دون مواصفة دفتر الحركات المعتمدة.

## قواعد البرمجة

- TypeScript صارم.
- لا تستخدم أرقام JavaScript العشرية مباشرة للحركات المالية؛ استخدم الوحدات الصغرى الصحيحة.
- لا تجمع SR وRG في قيمة أو حركة واحدة.
- كل عملية كتابة حساسة يجب أن تدعم منع التكرار وسجل التدقيق.
- لا حذف للحركات المعتمدة؛ استخدم الحركات العكسية.
- واجهات المستخدم عربية RTL ومتجاوبة.
- لا تعتمد على مزود استضافة واحد داخل منطق الأعمال.

## شروط الدمج

قبل دمج أي تغيير:

- نجاح lint.
- نجاح typecheck.
- نجاح build.
- نجاح الاختبارات عند إضافتها.
- تحديث الوثائق عند تغيير قاعدة عمل.
- إضافة ADR لأي قرار معماري جوهري.

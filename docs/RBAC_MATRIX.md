# مصفوفة الأدوار والصلاحيات – الإصدار الأولي

**الحالة:** معتمدة لوضع مدير الفرع الواحد، ومسودة للتوزيع متعدد المستخدمين  
**وضع التشغيل الحالي:** `SINGLE_MANAGER`

## قاعدة المرحلة الحالية

- مدير الفرع هو المستخدم التشغيلي الوحيد ويملك جميع الصلاحيات المعرفة حاليًا.
- يجوز له إنشاء العملية ومراجعتها واعتمادها بنفسه في `SINGLE_MANAGER`.
- تبقى كل مرحلة انتقالًا مستقلًا يسجل في سجل الحالات و`audit_logs`.
- لا يعني الاعتماد الذاتي السماح بتعديل الحركات المعتمدة أو حذفها؛ التصحيح بالعكس أو إصدار جديد.
- عند التحويل مستقبلًا إلى `MULTI_USER` تطبق قواعد الفصل الموضحة أدناه حسب الوحدة.

## الرموز

- نعم: مسموح ضمن نطاق السجل والحالة.
- لا: غير مسموح.
- قراءة: مشاهدة فقط.
- مشروط: يحتاج صلاحية إضافية أو اعتمادًا أعلى.

## قواعد نطاق السجل في MULTI_USER

- المندوب يرى العملاء والخطط والزيارات والتحصيلات التابعة له فقط.
- الحسابات ترى جميع التحصيلات والصندوق والمطابقات اللازمة لعملها.
- أمين المخزن يرى الأصناف والعهد والحركات المخزنية، ولا يرى الصندوق دون صلاحية مستقلة.
- مدير النظام لا يحصل تلقائيًا على بيانات العملاء أو الصلاحيات المالية لمجرد دوره التقني.
- المدقق يقرأ ولا يعدل.

## قواعد المرحلة

- لا يعدل المندوب التحصيل بعد إرساله إلا إذا أعيد إليه رسميًا.
- لا يعدل التحصيل بعد اعتماده؛ التصحيح بالعكس.
- لا تعتمد الخطة بعد بدء تنفيذها إلا عبر إصدار جديد موثق.
- لا تعدل المطابقة بعد الترحيل؛ ينشأ مسار تصحيح جديد.
- لا يفتح اليوم المغلق إلا بطلب واعتماد وإصدار إغلاق جديد.
- في `SINGLE_MANAGER` يجوز تطابق المنفذ بين المراحل، مع تسجيل `operating_mode` وسبب الاستثناء عند العمليات الحساسة.

## قرارات متبقية قبل MULTI_USER

- العمليات التي تتطلب موافقتين.
- حدود المبالغ التي ترفع الاعتماد إلى مدير الفرع.
- صلاحيات الحقول الحساسة مثل حد الائتمان والتكلفة.
- سياسة تجاوز مدير الفرع عند غياب المستخدم المختص.

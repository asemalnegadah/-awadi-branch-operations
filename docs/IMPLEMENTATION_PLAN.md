# خطة التنفيذ الرسمية

**النطاق:** نظام تشغيل ورقابة فرع العوادي – عدن فقط  
**المرجع الأعلى:** `docs/MASTER_SYSTEM_SPECIFICATION.md`

## المرحلة 0 — التأسيس والحوكمة

المخرجات:

- تهيئة المستودع.
- تثبيت قواعد المساهمة والتسمية.
- إعداد TypeScript والجودة والاختبارات وCI.
- منع الأسرار والبيانات الحقيقية من الدخول إلى GitHub.
- توثيق القرارات المعمارية ADR.

معيار الاكتمال:

- نجاح فحوص الجودة آليًا.
- وجود بيئة اختبار منفصلة.
- عدم وجود أسرار أو بيانات تشغيلية حقيقية في المستودع.

## قواعد تنفيذ عامة

- كل مرحلة لها معايير قبول واختبارات.
- لا يتم الدمج إلى الفرع الرئيسي عند فشل CI.
- لا بيانات حقيقية في Fixtures.
- لا أسرار في الكود.
- كل تغيير جوهري يوثق في ADR.
- لا تجاوز للمواصفة الرئيسية دون إصدار تعديل رسمي.

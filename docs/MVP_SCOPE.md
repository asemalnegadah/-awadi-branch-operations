# نطاق النسخة الأولى MVP

**الحالة:** مسودة تنفيذية قابلة للمراجعة  
**النطاق:** فرع العوادي – عدن فقط  
**المرجع الأعلى:** `MASTER_SYSTEM_SPECIFICATION.md`

## الهدف

إطلاق نواة تشغيلية موثوقة لإدارة العملاء، الديون، الوعود، الخطط، الزيارات، التحصيلات، تسليم النقدية، المطابقات والإغلاق اليومي، مع صلاحيات وتدقيق واستيراد آمن.

## المستخدمون المستهدفون

- المالك أو المراقب.
- مدير الفرع.
- الحسابات والصندوق.
- المندوبون.
- المدقق.
- مدير النظام.

يضاف أمين المخزن عند إدخال المخزون ضمن النواة أو المرحلة التالية.

## الوحدات الداخلة في MVP

1. تسجيل الدخول والجلسات.
2. المستخدمون والأدوار والصلاحيات.
3. إعدادات منشأة واحدة لفرع عدن.
4. المندوبون والمناطق وخطوط السير.
5. العملاء والهواتف والمواقع والمعرفات الخارجية.
6. تكليف العميل للمندوب وتاريخ التغييرات.
7. دفتر الحركات والأرصدة المنفصلة حسب العملة.
8. أعمار الديون.
9. الوعود ومتابعة الوفاء والكسر.
10. المخاطر ومنع الآجل والاستثناءات.
11. الخطط اليومية للمندوبين.
12. الزيارات والنتائج.
13. التحصيلات والسندات والإثباتات.
14. عهدة المندوب وتسليم النقدية والصندوق.
15. المطابقات والفوارق.
16. رفع الملفات والاستيراد والبوابات Gates 1–5.
17. جودة البيانات ومنع التكرار.
18. الإغلاق اليومي وإعادة الفتح.
19. لوحة التحكم والتقارير الأساسية.
20. سجل التدقيق والنسخ الاحتياطي والمراقبة الأساسية.

## الوحدات المؤجلة مبدئيًا

- المحاسبة العامة الكاملة.
- الرواتب والموارد البشرية.
- المشتريات وسلسلة التوريد الكاملة.
- إدارة فروع متعددة.
- تكامل مباشر مع أنظمة مصرفية.
- ذكاء اصطناعي ينفذ قرارات أو حركات.
- تحسين مسارات جغرافي متقدم يعتمد خرائط مدفوعة.

## المبيعات والمخزون

يصدر قرار إدخالهما في MVP بعد التحقق من مصدر الأرصدة الحالي:

- إن كانت الفواتير والمخزون هي المصدر المحاسبي الرسمي الذي لا يمكن بناء الرصيد دونهما، يدخل الحد الأدنى منهما في النواة.
- إن كانت البيانات الحالية تقدم أرصدة افتتاحية معتمدة، يؤجل البناء الكامل للمبيعات والمخزون إلى المرحلة التالية مع إبقاء قابلية الربط.

## حدود السعة الأولية

تثبت الأرقام النهائية في وثيقة السعة، والهدف المبدئي القابل للتوسع:

- حتى 10,000 عميل.
- حتى 50 مستخدمًا نشطًا.
- حتى 20 مستخدمًا متزامنًا.
- حتى 10,000 حركة تشغيلية يومية.
- استيراد ملفات حتى 50,000 صف عبر مهام خلفية.

هذه الحدود أهداف تصميم وليست وعودًا غير محدودة.

## شروط نجاح MVP

- فصل SR وRG تقنيًا ووظيفيًا.
- كل رصيد قابل للتتبع إلى دفتر الحركات.
- منع تعديل أو حذف الحركات المعتمدة مباشرة.
- منع الإرسال والاستيراد المكرر.
- المندوب لا يرى بيانات خارج نطاقه.
- تطابق الصندوق والتحصيلات وفق مراحل الاعتماد.
- استيراد تجريبي ومقارنة إجماليات قبل النقل النهائي.
- تقارير RTL موثقة بوقت قطع البيانات.
- نجاح نسخة احتياطية واستعادة تجريبية.
- عمل أهم وظائف المندوب تحت اتصال ضعيف.

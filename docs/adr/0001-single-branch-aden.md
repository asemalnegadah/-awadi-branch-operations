# ADR-0001: اعتماد نظام فرع عدن فقط

- **الحالة:** معتمد
- **التاريخ:** 2026-07-13
- **النطاق:** كامل النظام

## القرار

يبنى النظام لتشغيل ورقابة فرع العوادي في عدن فقط، دون دعم تعدد الفروع أو تعدد المستأجرين.

## النتائج الملزمة

- لا يستخدم `branch_id` في الجداول التشغيلية.
- لا يستخدم `tenant_id` أو `company_id` لعزل بيانات تشغيلية متعددة.
- لا توجد شاشة اختيار فرع.
- لا توجد صلاحيات حسب الفرع.
- لا توجد تقارير مقارنة فروع.
- لا توجد تحويلات مخزون أو صندوق بين فروع.
- توجد إعدادات منشأة واحدة داخل `organization_settings`.
- يسمح بعدة مخازن وصناديق ومندوبين ومناطق داخل فرع عدن.

## المبررات

- خفض التعقيد.
- تقليل أخطاء الصلاحيات والتقارير.
- تسريع نقل البيانات الحالية.
- تحسين الأداء وسهولة الاختبار.
- خفض تكلفة التشغيل والصيانة.

## ما لا يعنيه القرار

لا يمنع القرار تنظيم الكود بصورة جيدة وقابلة لإعادة الهندسة مستقبلًا، لكنه يمنع تحميل الإصدار الحالي أي منطق أو حقول أو واجهات لتعدد الفروع.

## معيار المراجعة

لا يغيّر هذا القرار إلا بطلب رسمي جديد، وتحليل أثر كامل على قاعدة البيانات والصلاحيات والتقارير والاستيراد، ثم ADR بديل معتمد.

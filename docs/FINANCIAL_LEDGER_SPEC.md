# مواصفة دفتر الحركات والأرصدة

**الحالة:** مسودة نواة مالية  
**القاعدة الأساسية:** الرصيد نتيجة حركات غير قابلة للمسح

## 1. الهدف

إنشاء نواة مالية قابلة للتتبع تمنع تعديل الرصيد يدويًا، وتفصل بين SR وRG، وتدعم الفواتير والتحصيلات والتخصيصات والحركات العكسية والأرصدة الافتتاحية.

## 2. مبادئ الدفتر

1. كل حركة لها معرف دائم غير قابل لإعادة الاستخدام.
2. كل حركة تخص عميلًا وعملة واحدة فقط.
3. كل حركة تحمل تاريخًا محاسبيًا ووقت إنشاء من الخادم.
4. الحركة المعتمدة لا تحذف ولا تعدل ماليًا.
5. التصحيح ينشئ حركة عكسية مرتبطة بالحركة الأصلية.
6. كل حركة تحمل مصدرها ومستخدمها وحالة اعتمادها.
7. لا يؤثر السجل المسودة في الرصيد الرسمي.
8. الرصيد الرسمي يحسب من الحركات المرحلة فقط.

## 3. أنواع الحركات الأولية

- رصيد افتتاحي مدين.
- رصيد افتتاحي دائن.
- فاتورة بيع.
- إشعار دائن.
- تحصيل.
- مرتجع.
- خصم معتمد.
- تسوية مطابقة معتمدة.
- حركة عكسية.

لا يضاف نوع جديد دون توثيق أثره على الرصيد والتقارير والاختبارات.

## 4. المراحل

المراحل العامة لحركة مالية:

`DRAFT → SUBMITTED → REVIEWED → APPROVED → POSTED`

المسارات الاستثنائية:

- `RETURNED`
- `REJECTED`
- `CONFLICTED`
- `REVERSED`

فقط `POSTED` تدخل في الرصيد الرسمي.

## 5. النموذج المقترح

### customer_ledger_entries

- `id`
- `customer_id`
- `currency_code`
- `entry_type`
- `direction`
- `amount_minor`
- `accounting_date`
- `status`
- `source_type`
- `source_id`
- `reversal_of_entry_id`
- `description`
- `created_at`
- `created_by`
- `approved_at`
- `approved_by`
- `posted_at`
- `posted_by`
- `request_id`
- `idempotency_key`

### invoices

رأس الفاتورة، العملة، العميل، المندوب، التواريخ، الحالة والإجماليات المحسوبة من البنود.

### collections

بيانات التحصيل والسند وطريقة الدفع والحالة، دون إنشاء أثر دفتر رسمي قبل الترحيل.

### invoice_allocations

ربط مبالغ التحصيل أو الإشعارات بالفواتير، مع منع تجاوز المتبقي إلا بمسار السداد الزائد.

### cash_movements

حركات عهدة المندوب والصندوق والتسليم والإيداع، منفصلة عن دفتر العميل لكنها مرتبطة بالتحصيل.

## 6. إشارة الحركة

يحدد `direction` الأثر:

- `DEBIT`
- `CREDIT`

لا يخزن المبلغ بقيمة سالبة؛ يخزن موجبًا وتحدد الإشارة من الاتجاه والنوع.

يجب اعتماد تعريف المدين والدائن لكل نوع حركة في جدول مرجعي واختبارات ثابتة.

## 7. التحصيل والتخصيص

يدعم التحصيل:

- تخصيص مباشر لفاتورة أو أكثر.
- تحصيل غير مخصص مؤقتًا.
- تخصيص جزئي.
- إعادة تخصيص قبل الترحيل وفق الصلاحية.
- سداد زائد يسجل كرصيد دائن أو دفعة مقدمة وفق السياسة المعتمدة.

لا يتجاوز مجموع التخصيصات مبلغ التحصيل.

## 8. الحركة العكسية

عكس حركة يتطلب:

- الحركة الأصلية مرحلة.
- سببًا إلزاميًا.
- صلاحية واعتمادًا مناسبًا.
- إنشاء حركة مساوية ومعاكسة في العملة نفسها.
- ربطًا متبادلًا بين الأصل والعكس.
- منع عكس الحركة أكثر من مرة دون مسار تصحيح جديد.

## 9. الأرصدة الافتتاحية

تنقل الأرصدة الحالية كدفعة افتتاحية معتمدة، ويجب أن تحمل:

- المصدر والصف.
- تاريخ القطع.
- العملة.
- المبلغ والاتجاه.
- دفعة الاستيراد.
- من راجع واعتمد.

لا تعدل بعد الترحيل؛ التصحيح بحركة تسوية معتمدة.

## 10. رصيد النظام ورصيد العميل والمطابقة

- **رصيد النظام:** ناتج الحركات المرحلة.
- **رصيد إفادة العميل:** قيمة يدلي بها العميل ضمن المطابقة ولا تغير الدفتر.
- **فرق المطابقة:** الفرق المحسوب بين القيمتين.
- **التسوية:** حركة مالية جديدة بعد اكتمال الاعتماد، وليست تعديلًا مباشرًا للرصيد.

## 11. منع التكرار

- كل طلب كتابة يحمل `idempotency_key` فريدًا ضمن نوع العملية.
- السندات لها قيود فريدة وفق العملة والسلسلة.
- الاستيراد يستخدم بصمة المصدر والصف.
- إعادة الطلب بالمعرف نفسه تعيد النتيجة السابقة ولا تنشئ حركة جديدة.

## 12. التتبع والتدقيق

كل حركة قابلة للتتبع إلى:

- المستخدم والجلسة.
- الطلب.
- المصدر والملف والصف.
- العملية التشغيلية الأصلية.
- المعتمد والمرحل.
- الحركة العكسية إن وجدت.
- إصدار قاعدة الحساب.

## 13. معايير القبول

- لا يوجد مسار API لتعديل رصيد العميل مباشرة.
- لا تدخل المسودة في الرصيد.
- لا تحذف الحركة المرحلة.
- عكس الحركة يعيد الأثر المالي بدقة.
- لا تختلط العملات في التخصيص.
- مجموع التخصيصات لا يتجاوز المبلغ المتاح.
- كل رصيد ظاهر يمكن شرحه بقائمة الحركات.
- إعادة نفس الطلب لا تنشئ حركة مكررة.
- تتساوى إجماليات الأرصدة الافتتاحية قبل النقل وبعده حسب العملة.

## 14. قرارات تحتاج اعتمادًا قبل التنفيذ النهائي

- المعنى المحاسبي الكامل لـ SR وRG.
- عدد المنازل العشرية النهائي.
- سياسة توزيع التحصيل غير المخصص.
- سياسة السداد الزائد.
- أنواع الخصومات والتالف والمرتجعات.
- لحظة تأثير التحصيل في دفتر العميل والصندوق.
- قواعد الترقيم الرسمية.

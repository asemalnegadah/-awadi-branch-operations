# تفعيل المدير واستعادة كلمة المرور بالبريد

## الهدف

يستخدم النظام رابطًا صالحًا لمرة واحدة لتفعيل حساب مدير الفرع أول مرة، ثم يستخدم المسار نفسه لاستعادة كلمة المرور مستقبلًا.

لا تحفظ كلمة المرور الخام أو رمز الاستعادة الخام في PostgreSQL أو GitHub أو سجل التدقيق. تحفظ فقط بصمة HMAC للرمز وبصمة Scrypt لكلمة المرور بعد اختيارها من المستخدم.

## متطلبات التشغيل

تضبط القيم التالية في مدير أسرار بيئة النشر، ولا تكتب في المستودع:

- `DATABASE_URL`
- `AUTH_SECRET`
- `APP_BASE_URL`
- `RESEND_API_KEY`
- `EMAIL_FROM`
- `ALLOW_INITIAL_MANAGER_EMAIL_BOOTSTRAP`
- `INITIAL_MANAGER_EMAIL`
- `INITIAL_MANAGER_NAME`

يجب أن يستخدم `APP_BASE_URL` بروتوكول HTTPS في الإنتاج.

## التفعيل الأول

1. تكون قاعدة البيانات بلا مستخدمين.
2. يضبط بريد المدير واسمه في متغيرات البيئة.
3. يفعل `ALLOW_INITIAL_MANAGER_EMAIL_BOOTSTRAP=true` مؤقتًا.
4. يفتح المدير صفحة `/forgot-password` ويكتب البريد المعتمد نفسه.
5. ينشئ النظام مستخدمًا بحالة `INVITED` ودور `BRANCH_MANAGER` ويرسل رابط تفعيل.
6. يفتح المدير الرابط ويختار كلمة مرور جديدة.
7. تتحول حالة المستخدم إلى `ACTIVE`، ويصبح الرابط مستهلكًا، وتلغى أي جلسات أو روابط أخرى.
8. بعد نجاح التفعيل يضبط `ALLOW_INITIAL_MANAGER_EMAIL_BOOTSTRAP=false` في بيئة النشر.

لا يستطيع بريد مختلف إنشاء المستخدم الأول، ولا يعمل التأسيس التلقائي إذا كان هناك أي مستخدم قائم.

## الاستعادة اللاحقة

1. يفتح المستخدم `/forgot-password`.
2. يعيد النظام دائمًا رسالة عامة لا تكشف وجود البريد من عدمه.
3. إذا كان الحساب مؤهلًا، يلغي النظام الرابط السابق وينشئ رابطًا جديدًا صالحًا لفترة قصيرة.
4. لا تحفظ قاعدة البيانات الرابط الخام.
5. عند تعيين كلمة المرور، تلغى كل الجلسات القديمة ويرتفع إصدار كلمة المرور.
6. الرابط مستعمل أو منتهي أو ملغى لا يقبل مرة أخرى.

## ضوابط الحماية

- حد لعدد الطلبات لكل بريد ولكل عنوان IP خلال ساعة.
- روابط عشوائية بطول 256 بت، تحفظ بصمتها فقط.
- الرابط يوضع في جزء `fragment` بعد `#` حتى لا يصل الرمز إلى سجل طلب الصفحة.
- صفحة الاستعادة تمنع الفهرسة وتستخدم سياسة `no-referrer`.
- طلبات الكتابة تتحقق من المصدر نفسه.
- إرسال البريد يستخدم مفتاح Idempotency لمنع التكرار.
- كل طلب وإرسال واستهلاك يسجل في `audit_logs` دون تسجيل كلمة المرور أو الرمز الخام.

## التحقق قبل الإنتاج

- تشغيل migrations من `0001` إلى `0013` على PostgreSQL نظيفة.
- تشغيل اختبارات قاعدة البيانات والتكامل.
- التحقق من نطاق الإرسال والبريد المرسل منه.
- إرسال رابط فعلي إلى بريد المدير واختبار التفعيل من البداية إلى تسجيل الدخول.
- التأكد من رفض إعادة استخدام الرابط.
- التأكد من إبطال الجلسات القديمة بعد الاستعادة.

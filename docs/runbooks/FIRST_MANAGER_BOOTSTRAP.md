# تشغيل حساب مدير الفرع الأول

## الغرض

إنشاء حساب مدير الفرع الأول بعد تطبيق migrations على قاعدة البيانات المستهدفة. لا ينفذ هذا الإجراء ضمن CI ولا يحتوي المستودع على كلمة مرور افتراضية.

## المتطلبات

1. تطبيق migrations حتى `0012_auth_single_manager.sql` بنجاح.
2. توفر Node.js 22 والاعتماديات المثبتة بواسطة `npm ci`.
3. تنفيذ الأمر من بيئة موثوقة تملك اتصالًا بقاعدة البيانات.
4. عدم كتابة أي قيمة حقيقية داخل GitHub أو ملف متتبع.

## متغيرات التنفيذ المؤقتة

- `DATABASE_URL`: اتصال قاعدة البيانات.
- `MANAGER_EMAIL`: البريد الذي سيستخدم لتسجيل الدخول.
- `MANAGER_NAME`: الاسم المعروض داخل النظام.
- `MANAGER_PASSWORD`: كلمة مرور طويلة وفريدة، من 12 إلى 128 حرفًا.
- `ALLOW_MANAGER_PASSWORD_RESET`: تبقى `false` عند الإنشاء الأول.

## الأمر

```bash
npm run bootstrap:manager
```

يقرأ الأمر القيم من بيئة العملية فقط. بعد نجاحه:

1. احذف `MANAGER_PASSWORD` من البيئة أو الجلسة الطرفية.
2. أبقِ `DATABASE_URL` و`AUTH_SECRET` داخل Secret Manager الخاص ببيئة التشغيل فقط.
3. سجل الدخول من صفحة `/login`.
4. افتح `/settings/security` لتغيير كلمة المرور عند الحاجة.

## الحماية

- يرفض الأمر إنشاء مدير ثانٍ ما دام النظام في وضع `SINGLE_MANAGER`.
- لا يعيد تعيين حساب موجود إلا عند ضبط `ALLOW_MANAGER_PASSWORD_RESET=true` صراحة.
- إعادة التعيين تبطل جميع الجلسات السابقة.
- يسجل الإنشاء أو إعادة التعيين في `audit_logs` دون حفظ كلمة المرور.
- لا يمنح الحساب صلاحيات قاعدة بيانات مباشرة؛ يحصل على دور التطبيق `BRANCH_MANAGER` فقط.

## الاستعادة عند نسيان كلمة المرور

نفذ الأمر نفسه بالبريد الحالي مع:

```text
ALLOW_MANAGER_PASSWORD_RESET=true
```

استخدم ذلك فقط من بيئة خادم موثوقة، ثم أعد القيمة إلى `false` واحذف كلمة المرور المؤقتة فورًا.

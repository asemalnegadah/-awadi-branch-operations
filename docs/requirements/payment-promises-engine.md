# محرك وعود السداد — المتطلبات التنفيذية

**النطاق:** فرع العوادي في عدن فقط  
**القرار المعماري:** ADR-0001  
**Migrations:** `0016_payment_promises_engine.sql` و`0017_payment_promises_cursor_precision.sql`

## حدود الإصدار

- جميع سجلات الوعود تخص منشأة وفرع عدن الوحيدين في النظام.
- لا يستخدم هذا الإصدار `branch_id` أو `tenant_id` ولا يقدم اختيار فرع أو تقارير مقارنة فروع.
- تعدد الفروع خارج نطاق الإصدار الحالي، ولا ينفذ مستقبلًا إلا بقرار ADR مستقل وتحليل أثر شامل للجداول المالية والصلاحيات والتقارير والاستيراد.
- العملات التشغيلية المسموحة هي `SR` و`RG` فقط، وتبقى كل عملة في سجلات وملخصات مستقلة.

## الكيانات

- `payment_promises`: رأس الوعد، حساب العميل، المندوب، العملة، المبالغ والتواريخ والحالة والإصدار.
- `payment_promise_events`: سجل أحداث غير قابل للتحديث أو الحذف.
- `payment_promise_followups`: متابعات غير قابلة للتعديل أو الحذف بعد تسجيلها.
- `payment_promise_allocations`: ربط الوعد بتحصيل مؤكد وعكس الربط دون حذف.

## الحالات

الحالات الأساسية المخزنة:

- `NEW`
- `UPCOMING`
- `PARTIALLY_FULFILLED`
- `FULFILLED`
- `REJECTED`
- `CANCELLED`

الحالتان `DUE_TODAY` و`OVERDUE` مشتقتان وقت القراءة بتوقيت `Asia/Aden`، ولا تخزنان كحقل يحتاج تحديثًا يوميًا.

## قواعد المبالغ

- جميع المبالغ أعداد صحيحة بالوحدات الصغرى.
- `fulfilled_amount_minor` لا يقبل الكتابة اليدوية؛ يعاد حسابه من التخصيصات الفعالة.
- `remaining_amount_minor` عمود محسوب من الموعود ناقص المنفذ.
- لا يتجاوز مجموع تخصيصات الوعد مبلغه الموعود.
- لا يتجاوز مجموع تخصيصات التحصيل مبلغ التحصيل.
- لا يربط إلا تحصيل بحالة `RECONCILED` أو `CLOSED` وله حركة دفتر مرتبطة ولم يُعكس.
- يجب تطابق العميل والحساب والعملة بين الوعد والتحصيل.
- تخصيص التحصيل وعكسه ينفذان داخل Transaction مع قفل صف الوعد ثم صف التحصيل بترتيب ثابت.
- العكس يوثق في السجل نفسه ويعيد حساب حالة الوعد؛ لا يحذف التخصيص.

## منع التكرار والتزامن

- الإنشاء والمتابعة والتخصيص والعكس والعمليات الحساسة تستخدم مفاتيح Idempotency فريدة.
- إعادة الطلب المطابق تعيد النتيجة السابقة، وإعادة المفتاح بطلب مختلف ترفض بتعارض.
- تحديث بيانات الوعد يتطلب `version` لمنع Lost Updates.
- ترتيب Cursor ثابت حسب `due_date, created_at, id`، وتطبّع دقة `created_at` إلى milliseconds لمنع إعادة آخر صف بين الصفحات.
- القيود والتريجرات في PostgreSQL تبقى خط الدفاع الأخير حتى عند الكتابة خارج خدمة التطبيق.

## الواجهة وAPI

تقدم الوحدة قائمة عربية RTL وفلاتر وPagination وبطاقات مستحق اليوم والمتأخر والمنفذ جزئيًا والمنفذ، وصفحات الإنشاء والتفاصيل والمتابعة والتخصيص والعكس والرفض والإلغاء والتصعيد. لا تعرض الواجهة إجراءً لا يملك المستخدم صلاحيته.

كل Write API يطبق المصادقة والصلاحية وSame-origin وRequest ID والتحقق وIdempotency وسجل الأحداث والتدقيق ورسائل الخطأ الآمنة.

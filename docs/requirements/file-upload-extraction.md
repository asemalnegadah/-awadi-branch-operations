# رفع الملفات واستخراج البيانات

## الهدف

تمكين المستخدم المصرح له من رفع ملفات تشغيلية، ثم استخراج المعلومات منها إلى منطقة مؤقتة للمراجعة قبل اعتمادها داخل النظام.

## أنواع الملفات المستهدفة

- PDF النصي.
- PDF المصور بعد OCR.
- الصور: PNG وJPG وWEBP.
- CSV وTSV.
- Excel: XLSX.
- النصوص: TXT وJSON.

## دورة الملف

1. `REGISTERED`: تسجيل بيانات الملف قبل الرفع.
2. `UPLOADED`: اكتمال تخزين الملف والتحقق من الحجم والبصمة.
3. `QUEUED`: إدخاله إلى طابور الاستخراج.
4. `EXTRACTING`: تشغيل المحلل المناسب حسب النوع.
5. `EXTRACTED`: حفظ النص والصفوف والحقول المستخرجة في منطقة مؤقتة.
6. `REVIEW_REQUIRED`: انتظار مراجعة بشرية وتصحيح التعارضات.
7. `APPROVED`: اعتماد نتيجة الاستخراج فقط.
8. `COMMITTED`: ترحيل السجلات المعتمدة إلى الجداول التشغيلية.
9. `REJECTED` أو `FAILED`: رفض الملف أو فشل المعالجة دون أي كتابة تشغيلية.

## قواعد غير قابلة للتجاوز

- رفع الملف لا يعني اعتماد بياناته.
- لا تكتب نتائج الاستخراج مباشرة في العملاء أو التحصيلات أو المبيعات أو المخزون.
- كل نتيجة تحمل مصدرها: الملف والصفحة والصف والخلية أو موضع النص.
- كل حقل مستخرج يحمل درجة ثقة وتحذيرات.
- التكرار يمنع ببصمة SHA-256 ومفتاح منع التكرار.
- الملفات لا تحفظ داخل GitHub.
- الملفات تحفظ في مزود تخزين خاص، وقاعدة البيانات تحفظ المرجع والبصمة والبيانات الوصفية فقط.
- لا يسمح بالتنفيذ أو المعاينة النشطة لأي محتوى مرفوع.
- يمنع SVG وHTML والملفات التنفيذية والملفات المضغوطة في المرحلة الأولى.
- حد الحجم الافتراضي 25 ميجابايت للملف، ويضبط من الإعدادات.
- فصل العملات SR وRG يطبق على كل مبلغ مستخرج.
- لا يعتمد أي سجل منخفض الثقة أو متعارض آليًا.

## مخرجات الاستخراج

- نص خام منظم حسب الصفحة أو الورقة.
- جداول وصفوف مستخرجة.
- حقول قياسية مثل العميل، الرقم، الهاتف، المندوب، العملة، المبلغ، التاريخ، رقم السند، الفاتورة، الصنف والكمية.
- حقول غير معروفة تحفظ كما هي للمراجعة ولا تهمل.
- أخطاء وتحذيرات وتعارضات.
- سجل تدقيق كامل لكل مراجعة واعتماد وترحيل.
